# 题目 + 技术栈 + 目标 + 规划

## 一、题目
基于大模型与 Git 协作的 Unity 工程代码实时检测、多轮评估与智能通知系统

## 二、技术栈
- 代码仓库与协作：GitHub（或 Gitee）
- 事件接入：Webhook
- 后端服务：Python + FastAPI
- 异步任务：Celery + Redis
- 数据存储：PostgreSQL
- 规则检查：Roslyn 分析器 + StyleCop（C#）
- 大模型评估：OpenAI API（或同类可替换模型接口）
- 通知渠道：企业微信机器人或飞书机器人
- 可视化面板：Vue3 + Element Plus
- 部署方式：Docker Compose

## 三、目标
- 对每次提交或 PR 的代码变更进行自动检测与评估。
- 将规则检测和大模型语义评估组合为多轮评估流程。
- 输出可追踪的质量评分、风险项与改进建议。
- 在触发阈值时自动发送分级通知给项目成员。
- 提供简洁看板展示趋势、问题分布与处理结果。

## 四、总体规划
- 建立“事件接入 -> 任务分发 -> 多轮评估 -> 结果聚合 -> 通知与展示”的闭环。
- 先完成最小可用版本，再逐步补充评估准确度与展示能力。
- 重点保证可演示、可度量、可复现，满足学期答辩要求。
